## 2. 小程序端页面设计规格

### 2.1 扫码落地页

#### 2.1.1 页面概述

访客扫描项目二维码后的第一个页面,用于选择访客类型并进入登记流程。

#### 2.1.2 页面布局

```
┌─────────────────────────────────┐
│      导航栏                      │
│  到访信息快速登记                 │
├─────────────────────────────────┤
│                                 │
│    ┌───────────────────────┐   │
│    │  访客类型选择弹窗       │   │
│    │                       │   │
│    │  [探亲访友] [家政维修] │   │
│    │  [房屋中介] [外卖]     │   │
│    │  [快递]               │   │
│    │                       │   │
│    │  [关闭]               │   │
│    └───────────────────────┘   │
│                                 │
└─────────────────────────────────┘
```

#### 2.1.3 组件清单

| 组件名称 | 类型 | 数量 | 说明 |
|---------|------|------|------|
| 导航栏 | 自定义 | 1 | 标题"到访信息快速登记",左侧返回按钮 |
| 访客类型弹窗 | 自定义 | 1 | 底部弹出,5种访客类型选项 |
| 访客类型卡片 | 自定义 | 5 | 图标+文字,点击选择 |

#### 2.1.4 视觉设计

**访客类型弹窗**:
- 背景: 白色(#FFFFFF)
- 圆角: 12px(顶部)
- 阴影: 0 -4px 16px rgba(0, 0, 0, 0.08)
- 内边距: 24px

**访客类型卡片**:
- 尺寸: 90px × 90px
- 背景: 白色(#FFFFFF)
- 边框: 1px solid #E5E7EB
- 圆角: 8px
- 图标尺寸: 48px × 48px
- 文字大小: 14px
- 文字颜色: #1F2937
- 间距: 16px

**访客类型图标**:
- 探亲访友: 家庭图标
- 家政维修: 工具图标
- 房屋中介: 房屋图标
- 外卖: 餐饮图标
- 快递: 包裹图标

#### 2.1.5 交互设计

**初始加载**:
1. 页面加载完成后,自动弹出访客类型选择弹窗
2. 弹窗从底部滑入,动画时长 300ms

**选择访客类型**:
1. 点击访客类型卡片,卡片边框变为主色(#2563EB)
2. 弹窗收起,动画时长 200ms
3. 根据访客类型跳转到对应的登记页面

**关闭弹窗**:
1. 点击关闭按钮或遮罩层,弹窗收起
2. 默认进入"探亲访友/房屋中介/家政维修"类型表单

#### 2.1.6 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 二维码已禁用 | 显示提示页"该二维码已停用,请联系物业",不允许登记 |
| 二维码已过期 | 显示提示页"该二维码已失效,请联系物业更新" |
| 网络异常 | 显示 Toast 提示"网络异常,请检查网络后重试" |

| 字段名 | 类型 | 必填 | 校验规则 | 说明 |

- 内边距: 0 16px
- 标签字体: 14px, #1F2937
- 输入字体: 14px, #1F2937
- 占位符字体: 14px, #9CA3AF
- 必填标记: 红色(#D9001B)星号

**提交按钮**:
- 背景: 主色(#2563EB)
- 高度: 48px
- 圆角: 8px
- 字体: 16px, 白色(#FFFFFF), 居中
- 内边距: 0 16px
- 禁用状态: 背景 #D1D5DB

**分组标题**:
- 字体: 14px, #6B7280
- 上边距: 24px
- 下边距: 12px
- 左内边距: 16px

#### 2.2.7 交互设计

**房号选择器**:
1. 点击房号字段,弹出级联选择器
2. 选择顺序: 楼栋 → 单元 → 房号
3. 支持搜索功能
4. 房号数据从 ERP 实时获取

**表单验证**:
1. 实时验证: 输入时验证格式
2. 提交验证: 点击提交按钮时验证必填项
3. 错误提示: 字段下方显示红色错误文字

**提交表单**:
1. 点击"生成通行码"按钮
2. 按钮显示 loading 状态
3. 提交成功后跳转到通行码页面
4. 提交失败显示错误提示

#### 2.2.8 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 一键登录失败 | 显示提示"获取手机号失败,请手动输入",提供手动输入选项 |
| ERP房间数据获取失败 | 显示提示"获取房间信息失败,请稍后重试",提供刷新按钮 |
| 网络异常 | 保留已填写信息,显示重新提交按钮 |
| 访客重复登记 | 显示提示"您已有有效通行码,是否查看?",提供查看或重新登记选项 |
